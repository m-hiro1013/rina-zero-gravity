# 技術選定ルール (Tech Selector)

プロジェクトタイプに応じて最適な技術スタックを選定するよ！

## 選定フロー

```
Step 1: ユーザーの要求を分析
    ↓
Step 2: キーワードからタイプを推定
    ↓
Step 3: 3つの選択肢を提案
    ↓
Step 4: ユーザーに選択してもらう
    ↓
Step 5: 技術スタックを確定
```

## 判定ロジック

### キーワード → プロジェクトタイプ

| キーワード | プロジェクトタイプ | 推奨テンプレート |
|-----------|------------------|-----------------|
| 「アプリ」「ダッシュボード」「管理画面」 | Webアプリ | web-app |
| 「サイト」「ポートフォリオ」「シンプル」 | シンプルサイト | simple-site |
| 「LP」「ランディングページ」「セールス」 | ランディングページ | landing-page |
| 「API」「サーバー」「バックエンド」 | APIサーバー | api-server |
| 「データ分析」「可視化」「グラフ」 | データダッシュボード | data-dashboard |
| 「拡張機能」「Chrome」「ブラウザ」 | Chrome拡張 | chrome-extension |

### 技術的要件 → 追加判定

| 要件 | 推奨技術 |
|------|---------|
| 認証が必要 | Supabase Auth / NextAuth |
| リアルタイム性 | Supabase Realtime / WebSocket |
| SEO重要 | Next.js（SSR/SSG） |
| 静的で十分 | HTML/CSS/JS |
| データベース必要 | Supabase / PostgreSQL |
| 高速API | FastAPI |
