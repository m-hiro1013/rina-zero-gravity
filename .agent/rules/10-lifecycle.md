# プロジェクトライフサイクル (Project Lifecycle)

プロジェクト全体の大きな流れを定義する。
AIによる開発の強みは、**「超小さいサイクル」を無限に回して、確実に完成度を高めること** にある。

## 全体の流れ（マクロ）

```
Phase 0: 要件定義（Define）     ← 何を作るか「合意」する
    ↓
Phase 1: 環境構築（Setup）      ← 爆速で「足場」を固める
    ↓
Phase 2: 反復開発（Iterate）    ← 本番！最小単位で実装・テストを繰り返す ♻️
    ↓
Phase 3: デプロイ（Deploy）     ← 完成したものを「公開」する
    ↓
Phase 4: 保守（Maintain）       ← 継続的に良くしていく
```

## AI駆動開発の核心：反復サイクル ♻️

このプロジェクトでは、Phase 2 の「反復開発」をどれだけ高い精度で回せるかが重要である。
具体的なフェーズの定義や、エージェントごとの役割は以下のルールを参照すること。

- **詳細なフェーズ定義**: `@rules/11-phase-definitions.md`
- **エージェントの割り当て**: `@rules/12-agent-assignment.md`

## フェーズ遷移と進捗管理

### 基本ルール
1. 各サイクルの成果は **`prompt/WORKFLOW.yaml`** に保存する。
2. 「今どのフェーズ？」「次何する？」は常に可視化しておく。
3. 2回以上同じミスを繰り返したら、Phase 2 を中断してルールを見直す（自己成長）。

## ユーザーへの報告

### フェーズ開始時
```
Phase X: {{フェーズ名}} を開始
```

### サイクル完了時
```
サイクル 1-1 完了。
WORKFLOW.yaml を更新済み。次のステップへ進行可能。
```

## 緊急時の対応

### バグ発見時
- `/bug-fix` ワークフローを起動して、反復サイクルの中で即座に修正する。

### 要件変更時
- Phase 0（要件定義）に一度戻って、`PROJECT_SPECIFIC.yaml` を同期し直す。
