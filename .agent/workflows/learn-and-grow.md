# /learn-and-grow

> 戦いの中で進化する。知見を蓄積し、次に活かす。

## 概要

このワークフローは **GrowthMonitor Agent** の中核機能。
セッションで得た学びを体系化し、プロジェクトの知恵として蓄積する。

## トリガー

- セッション終了時（/save-session から呼び出し）
- 明示的な `/learn-and-grow` 実行

## 実行手順

### Step 1: セッション振り返り

今回のセッションを振り返る：

```
📝 今回のセッションで...

1. **うまくいったこと**は何？
2. **ハマったこと**は何？
3. **新しく学んだこと**は何？
4. **次回に活かせること**は何？
```

### Step 2: 知見の抽出

以下の観点で知見を抽出する：

| 観点 | 質問 |
|------|------|
| 環境構築 | セットアップで困ったことは？ |
| 設計パターン | 良かった設計、悪かった設計は？ |
| ハマりポイント | 時間がかかった問題は？ |
| ベストプラクティス | 「これは使える！」と思ったことは？ |

### Step 3: ウェイト判断

各知見にウェイトを付与する：

```
+3 = 必須級。これ無いと事故る
+2 = 推奨。あると品質上がる
+1 = 参考。状況や好みによる
```

**判断基準:**
1. 権威性 - 公式ドキュメントか、実体験か
2. 現実性 - 何回実証されたか
3. 信頼性 - 再現可能か

### Step 4: KNOWLEDGE.md 更新

// turbo
```bash
# prompt/KNOWLEDGE.md を開いて追記
```

追記フォーマット:
```markdown
### {{カテゴリ}}

- {{知見の内容}} [+{{ウェイト}}] ({{日付}})
```

### Step 5: セッションログ追記

// turbo
```markdown
### Session: {{DATE}}

**学んだこと:**
- {{学んだこと1}}
- {{学んだこと2}}

**改善点:**
- {{改善点1}}
```

### Step 6: 統合判断

**統合基準（親プロジェクトに持っていくか）:**

| 条件 | 統合する？ |
|------|-----------|
| このプロジェクト固有の知見 | ❌ No |
| 汎用的に使える知見 | ✅ Yes |
| ウェイト +3 の重要知見 | ✅ Yes |
| 他プロジェクトでも再発しそう | ✅ Yes |

統合する場合:
1. `rina-zero-gravity/prompt/references/goku.md` を参照
2. 適切なカテゴリに追記
3. 統合履歴を KNOWLEDGE.md に記録
