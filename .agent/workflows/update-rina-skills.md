# /update-rina-skills - Rina進化ワークフロー

> 戦いの中で進化する。エージェントが自分自身を成長させる。

## 概要

このワークフローは、子プロジェクトで得た知見を
親プロジェクト（rina-zero-gravity）に統合し、
Rina自身のスキルをアップデートする。

## トリガー

- 明示的な `/update-rina-skills` 実行
- `/learn-and-grow` で汎用知見が特定された時
- 新しいワークフロー/ルールのパターンが確立された時

## 前提条件

- 親プロジェクト（rina-zero-gravity）へのアクセス権
- 統合対象の知見が明確に特定されている

## Step 1: 統合候補の特定

子プロジェクトの `prompt/KNOWLEDGE.md` から、以下の条件を満たす知見を抽出:

| 条件 | 統合する？ |
|------|-----------|
| このプロジェクト固有の知見 | ❌ No |
| 汎用的に使える知見 | ✅ Yes |
| ウェイト +3 の重要知見 | ✅ Yes |
| 他プロジェクトでも再発しそう | ✅ Yes |

```
🔍 統合候補を探すね！

【今回のプロジェクトで得た知見】
- {{知見1}} [+{{ウェイト}}]
- {{知見2}} [+{{ウェイト}}]

【統合候補】
- {{汎用知見1}} [+{{ウェイト}}] ← これ使える！
```

## Step 2: 抽象化

個人情報・プロジェクト固有情報を除去して抽象化:

```
❌ 除外するもの:
- ユーザー名・本名
- プロジェクト名・サービス名
- URL・ドメイン
- APIキー・トークン
- ファイルパス（プロジェクト固有）

✅ 抽象化例:
- 「〇〇さんのプロジェクト」→「実践で学んだ」
- 「ABCサービスのAPI」→「外部API」
```

## Step 3: 統合先の決定

| 知見の種類 | 統合先 |
|-----------|--------|
| 技術的なtips | `prompt/references/goku.md` |
| 新しいワークフロー | `.agent/workflows/` |
| 新しいルール | `.agent/rules/` |
| テンプレート改善 | `.agent/templates/` |

## Step 4: 統合実行

// turbo
```bash
cd /Users/matsumotohiroki/Developer/rina-zero-gravity
```

### 4-1: goku.md への知見追加

```markdown
### {{カテゴリ}}

- {{抽象化された知見}} [+{{ウェイト}}] ({{日付}})
```

### 4-2: ワークフローの追加（該当する場合）

新しいワークフローパターンが確立された場合:

1. `.agent/workflows/` に新規ファイル作成
2. 既存ワークフローと整合性確認
3. ワークフロー一覧を更新

### 4-3: ルールの追加（該当する場合）

新しいルールが必要な場合:

1. `.agent/rules/` に新規ファイル作成
2. 番号プレフィックスで優先順位を設定
3. 既存ルールと矛盾がないか確認

## Step 5: 統合履歴の記録

子プロジェクトの `prompt/KNOWLEDGE.md` に統合履歴を記録:

```markdown
## 統合履歴

| 日付 | 統合先 | 内容 |
|------|--------|------|
| {{日付}} | goku.md | {{知見の概要}} |
| {{日付}} | workflows/ | {{ワークフロー名}} |
```

## Step 6: 検証

統合後の整合性を確認:

- [ ] goku.mdにsyntaxエラーがないか
- [ ] 追加したワークフローが動作するか
- [ ] 既存機能に影響がないか

## 注意事項

⚠️ **アイデンティティの原則**
技は吸収する。魂は売らない。

- ✅ 取り込むもの: 具体的なテクニック、設計パターン、ベストプラクティス
- ❌ 取り込まないもの: 口調・話し方、性格・キャラ設定、価値観の上書き

⚠️ **プライバシーの原則**
知見は共有する。個人情報は絶対に共有しない。
